階段一：API 資料整理與 Excel 轉換

核心檔案：convert_to_excel.py · 2026-07-03
內容：

從指定 API 抓取博弈遊戲資料，將 JSON 回傳內容整理成清楚易讀的格式，並輸出成 Excel 報表。 同時將原始英文欄位名稱翻譯為繁體中文（例如 gameName → 遊戲名稱、profit → 玩家輸贏）， 並自動計算 RTP（返回率）、調整欄位排序與凍結首列，讓報表更符合營運查看習慣。

  • 支援多個 API 來源同時合併（以空格或逗號分隔）
  • 支援本地 JSON 檔案作為資料來源
  • 數值欄位自動清洗（移除逗號、處理空值）
  • 結算時間降冪排序，最新資料排在最上方
pandas openpyxl requests

階段二：風控追殺局分析

整合於 convert_to_excel.py
內容：

依照風控規則，分析各遊戲在追殺模式（防守局）中玩家破殺的機率。 追殺局代碼為 ptk、K、T、B， 若玩家在追殺局中贏錢（玩家輸贏 > 0）視為「破殺」。 當破殺率超過 5% 時，於終端機輸出 🚨 風控警示訊息。

  • 依遊戲名稱分組統計總局數、追殺局數、破殺率
  • 額外產出「追殺局分析」Excel 報表
  • 可選：同步資料與分析結果至 Google Sheet（含堆疊柱狀圖）
風控 gspread Google Sheet

階段三：Allure 測試報表自動化

核心檔案：test_risk_control.py · run_allure.py
內容：

在完成資料整理後，針對整體流程進行 pytest 測試驗證，確認 API 資料抓取、欄位轉換、 風控分析與異常判斷皆可正常運作。測試結果透過 Allure 報表呈現， 每一款遊戲對應一個測試案例，可清楚看到測試名稱、成功/失敗狀態與詳細數據附件。

  • 動態生成測試案例（依 API 回傳的遊戲數量）
  • 自動附加 Excel 報表至 Allure 報告
  • 破殺率 > 5% 的遊戲標記為測試失敗
  • 產生單一 HTML 靜態報告，可直接分享
pytest allure-pytest Java

階段四：一鍵執行器與報告分享

核心檔案：run_allure.py
內容：

整合上述所有功能為一鍵執行器：輸入分析名稱與 Google Sheet 資訊後， 自動執行 pytest、產生 Allure 報告、啟動本地 Web 伺服器並在瀏覽器開啟。 另可透過 Cloudflare Tunnel 產生臨時外網分享網址，方便傳送報告給他人。

  • 自動偵測並載入 Java 環境（免重啟終端機）
  • 優先使用專案 .venv 虛擬環境
  • 動態尋找可用 Port 部署本地伺服器
  • 外網網址自動複製至剪貼簿
cloudflared webbrowser subprocess

階段五：API 設定安全化

核心檔案：api_config.py · .env
內容：

將原本寫死在程式碼中的內網 API 位址移至 .env 環境設定檔， 避免上傳 GitHub 時暴露內網 IP。程式啟動時自動讀取 .env， 終端機顯示時遮罩 host 位址，確保敏感資訊不外洩。

  • .env.example 作為範本（可安全上傳 GitHub）
  • .env 加入 .gitignore（本機專用）
  • 未設定時可手動輸入 API，不寫入程式碼
安全 .env gitignore

遠端倉庫：leolin-ship-it/TestProject

使用 Git 進行版本控管，將專案推送至 GitHub 公開倉庫。 透過 .gitignore 排除虛擬環境、測試報告、憑證檔與本機設定， 確保只有程式碼與文件上傳，敏感資料留在本機。

  • 初始化 repo 並完成首次 commit

Git GitHub 版本控管

階段七：GitHub Pages 風控分析儀表板

核心目錄：docs/ · 2026-07-06
內容：

因 GitHub Pages 只能托管靜態網頁、無法執行 Python 或存取內網 API， 將風控分析核心邏輯移植為純 JavaScript，建立可在瀏覽器運行的分析儀表板。 使用者上傳 JSON 檔案即可在網頁上看到完整的追殺局分析、圖表與風控警示， 資料完全在本地運算，不上傳至任何伺服器。

  • 拖放上傳 JSON、Chart.js 堆疊柱狀圖
  • 破殺率 > 5% 標示異常、可下載 HTML / JSON 報告
  • GitHub Actions 自動部署至 Pages
  • 內建範例資料供快速體驗
HTML JavaScript Chart.js GitHub Pages

階段八：本機 JSON 匯出橋接

核心檔案：scripts/export_for_web.py
內容：

建立橋接腳本，在本機從內網 API 匯出 JSON 檔案， 再上传至 GitHub Pages 分析儀表板進行分析。 解決「內網 API 無法在公開網頁直接存取」的問題， 讓本機資料安全地流向網頁分析工具。

  • 讀取 .env 中的 API 位址
  • 合併多來源資料並匯出至 docs/exports/
  • 匯出目錄加入 .gitignore，避免真實資料上傳
橋接 JSON 匯出 內網 → 網頁
